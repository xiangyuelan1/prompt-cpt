# 提示词竞赛平台 - 产品需求文档

## 1. 产品愿景与目标

### 1.1 产品定位
打造国内首个专注于AI提示词优化的竞技平台，让用户通过竞赛形式提升提示词编写能力，同时挖掘和分享优质提示词模板。

### 1.2 核心价值
- **竞技性**：通过挑战赛制激发用户参与热情
- **学习性**：通过实时预览和优秀模板提升用户技能
- **社区性**：通过投票和互动建立活跃的提示词创作者社区

### 1.3 目标用户
- AI爱好者：希望提升与AI工具交互效率的普通用户
- 提示词创作者：专注于优化和分享高质量提示词的专业用户
- AI工具用户：需要针对特定场景（如写作、代码、图像等）获得优质提示词的用户

## 2. 核心功能需求

### 2.1 竞赛系统

#### 2.1.1 挑战赛制
- **主题挑战**：每周/每日发布特定主题的挑战（如"最佳小说开头提示词"、"最实用的代码注释提示词"等）
- **实时排名赛**：竞赛期间随时提交，系统实时计算并更新排名
- **挑战周期**：每场挑战持续24-72小时，到期自动截止

#### 2.1.2 参赛流程
1. 用户浏览当前进行的挑战列表
2. 选择感兴趣的挑战主题
3. 在提示词编辑器中编写参赛作品
4. 通过实时预览功能测试效果
5. 提交作品参赛
6. 竞赛结束后查看最终排名和评分

### 2.2 评分系统

#### 2.2.1 混合评分机制
- **AI自动评分（60%权重）**
  - 输出质量评估：基于提示词生成的AI输出效果评分
  - 结构评估：提示词的格式规范性、关键词使用、逻辑清晰度
  - 创意评估：独特性、创新性、实用性

- **社区投票（40%权重）**
  - 其他用户对作品进行点赞和评分
  - 设置投票截止时间，与AI评分结合计算最终成绩

#### 2.2.2 评分维度细化
| 维度 | 权重 | 说明 |
|------|------|------|
| 清晰度 | 20% | 指令是否明确、无歧义 |
| 创意性 | 20% | 是否具有独特思路和创新点 |
| 实用性 | 30% | 在实际使用中的效果和泛化能力 |
| 社区投票 | 30% | 用户评价和认可度 |

### 2.3 实时预览系统

#### 2.3.1 预览功能
- **即时反馈**：用户在编辑提示词时，实时显示AI模拟输出效果
- **多场景测试**：支持针对不同AI模型和应用场景进行预览
- **参数调整**：可调整生成参数（长度、风格、格式等）

#### 2.3.2 预览配置
- 默认使用通用AI模型模拟输出
- 支持展示输出效果的可读性展示
- 提供预览历史的快速切换

### 2.4 排行榜系统

#### 2.4.1 实时排名
- 竞赛期间实时更新排名
- 显示当前排名、分数变化趋势
- 展示TOP10选手的提示词片段

#### 2.4.2 历史排行
- 往期挑战的最终排名
- 用户综合积分排行榜
- 各类别（写作、代码、图像等）的专项排行

### 2.5 成就系统

#### 2.5.1 成就徽章
- **参赛成就**：完成首次参赛、连续参赛等
- **排名成就**：进入TOP10、获得冠军、多次获奖等
- **质量成就**：提示词获得高评分、高投票等
- **社区成就**：被收藏次数多、获得大量点赞等

#### 2.5.2 积分系统
- 参赛获得基础积分
- 排名靠前获得额外积分
- 作品被收藏、被复用获得积分
- 积分可兑换平台特权或虚拟奖励

### 2.6 提示词模板库

#### 2.6.1 模板来源
- 竞赛获奖作品自动收录
- 用户主动提交优质提示词
- 管理员精选推荐

#### 2.6.2 模板管理
- 分类浏览：按用途（写作、代码、图像、分析等）
- 标签检索：支持多标签筛选
- 一键使用：快速复制并应用于新竞赛
- 收藏功能：用户收藏喜欢的模板

## 3. 用户交互流程

### 3.1 首页交互
```
首页展示：
├── 当前进行的挑战（卡片式展示）
├── 实时排行榜（TOP5快速预览）
├── 最近获奖提示词展示
└── 快捷入口：开始新挑战、浏览模板库
```

### 3.2 竞赛参与流程
```
选择挑战 → 阅读挑战要求 → 编写提示词 → 实时预览测试 → 提交作品 → 查看排名
```

### 3.3 模板使用流程
```
浏览模板库 → 筛选/搜索 → 查看详情 → 收藏或直接使用 → 进入编辑器
```

## 4. 界面设计方向

### 4.1 整体风格
- **科技感**：融合AI/代码元素的现代科技风格
- **竞技感**：突出排名、竞争、成就等元素
- **简洁高效**：编辑器界面专注、高效

### 4.2 色彩方案
- **主色调**：深蓝/紫色系（代表科技和AI）
- **强调色**：渐变金色/橙色（代表竞赛和成就）
- **功能色**：青绿色（代表成功、预览通过）
- **警示色**：红色系（用于时间紧迫、排名下降等）

### 4.3 布局特点
- **首页**：信息密度适中，突出重点挑战和排名
- **编辑器**：全屏沉浸式，左右分栏（编辑/预览）
- **排行榜**：动态数据展示，带有动画效果
- **模板库**：卡片网格布局，支持快速预览

### 4.4 交互动效
- 排名变化时带有平滑的过渡动画
- 提交成功时带有成就感的反馈动画
- 实时预览区支持打字机效果的模拟输出
- 成就获得时带有徽章解锁动画

## 5. 技术需求

### 5.1 前端技术栈
- React + TypeScript
- Tailwind CSS 进行样式开发
- 状态管理：React Context 或 Zustand
- 动画库：Framer Motion
- 代码编辑器组件（用于提示词编辑）

### 5.2 模拟AI功能
由于是前端演示项目，AI评分和实时预览将通过模拟实现：
- **AI评分模拟**：基于预设规则生成随机但合理的评分
- **实时预览模拟**：基于提示词关键词生成模拟输出文本
- **数据结构**：完整定义接口数据结构，便于后续对接真实API

### 5.3 数据存储
- 本地存储（LocalStorage）用于演示
- 支持用户数据、竞赛数据、模板库的持久化
